About Me

저는 이런 사람입니다.

효율적인 업무 처리를 중시하는 김혜지입니다.

업무를 구조화하고 스케줄링하여
능률적인 진행을 추구하며,
커뮤니케이션 능력으로 팀원들과 협업 시
원활한 소통을 유지하며 작업을 완수한 경험이 있습니다.

함께할 때는 팀원들과 조화롭게 일하며,
혼자일 땐 냉정하게 문제를 해결하며
효과적인 결과를 창출하여 성장하겠습니다.

Tools & Library

저는 이것을 주로 사용합니다.

AWS

AWS 아키텍처를 구성한 뒤,
직접 생성하고 연결할 수 있습니다.

보안을 위해 VPC 및 서브넷을 생성하고,
역할과 정책을 구성할 수 있습니다.


GCP

GCP 아키텍처를 구성한 뒤,
직접 생성하고 연결할 수 있습니다.

모니터링을 위해 대쉬보드 및 로깅을 구성하고,
Big Query와 Looker를 이용해
데이터 분석 및 시각화를 할 수 있습니다.

Kubernetes

Kubernetes 클러스터를 구성하고, 파드, 서비스, 볼륨 등의 리소스를 관리할 수 있습니다.

Docker 컨테이너 기술을 이용하는 애플리케이션을 구성하고 배포해 본 경험이 있습니다.

Linux

Linux 커맨드 라인 인터페이스(CLI)를 이용하여
파일 및 디렉토리 작업, 패키지 설치, 서비스 제어
등을 수행할 수 있습니다.

Linux에서 사용되는 Ubuntu를 설치하고
구성하는 방법을 알고 있습니다.

Terraform

Terraform을 사용하여 IaC(Infrastructure as Code)를 구현하고, 인프라스럭처의 버전 관리 및
변경 이력 추적을 할 수 있습니다.

Terraform을 이용하여 각 CSP의 서비스를
구성하고 프로비저닝할 수 있습니다.

Visual Studio Code

VS Code를 사용하여 코드 편집, 버전 관리,
플러그인 관리 등 다양한 작업을
수행할 수 있습니다.

VS Code의 단축키와 명령어를 이용하여 더욱
효율적으로 코드 작업을 수행할 수 있습니다.

Boutique
& DR

[ GCP, MSA, Terraform, CICD ]
쇼핑몰 & 재해복구 시스템


Summary

Prod & dev 환경의 다중 클러스터 구조의 온라인 부티크 서비스 생성 및
재해 상황 시 테라폼 코드를 이용한 서비스 복구를 통해
고객에게 원활한 서비스를 제공하는 것이 목표입니다.

사용자는 페이지 접속 후, 구매할 물건의 선택 후 장바구니에 추가합니다.
해당 정보는 세션이 유지되어 사용자가 웹을 다시 켜도 유지됩니다.

데이터베이스는 GCP를 이용하여 구축했으며,
데이터 백업 설정을 통해 재해 상황을 대비했습니다.
사전에 설정한 알림, 모니터링을 통해 재해 발생을 인지하여
DRS을 실행시켜 서비스를 복구합니다.
